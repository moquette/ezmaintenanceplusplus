v2026.07.19.4

v2026.07.19.3
- Internal: the add-on now clears its own stale compiled-code cache at startup.
  Python can keep running an older compiled copy of a file when the new one is
  the same size, and because builds here use a fixed timestamp for
  reproducibility, that check could not tell the two apart. Clearing the cache
  means the code that runs is the code that shipped. No change to anything you
  see or use.

v2026.07.19.1
- Internal: the add-on now reports a fingerprint of its own installed
  storage-handling code, and the release check compares that against the code
  being certified. Previously the check trusted the version number, which does
  not change when that code does, so a box running an older build could be
  recorded as verified. No change to anything you see or use.

v2026.07.19.0
- A restore that switches to a different skin now tells you. Kodi only applies a
  skin change when someone confirms it on screen, and that confirmation cannot be
  answered safely from inside a restore, so the box reopens on the skin it was
  already running. The restored skin and all of its settings are installed and
  intact - nothing is lost - but the switch has to be finished by hand in
  Settings > Interface > Skin. The next start now checks for this and says so
  instead of leaving it to be noticed. Restores that do not change the skin, which
  is the normal case, are unaffected and stay silent.

v2026.07.18.2
- Backing up to a location inside Kodi's own storage now warns first. On Apple
  TV everything Kodi owns lives in an area the system may clear to free space,
  and a wipe or reinstall removes it, so a backup saved there can be gone at the
  moment it is needed. The browse dialog offers that location as its starting
  point, which made it the easiest one to pick by accident. USB, network and
  Dropbox destinations are unaffected and are not warned about.

v2026.07.18.1
- Restoring a skin setting whose value contains a comma no longer corrupts it.
  Values are passed to Kodi's Skin.SetString builtin, and Kodi splits builtin
  parameters on commas, so a label like "Movies, HD" arrived truncated to
  "Movies". Because the truncated value was set in live memory, Kodi's
  clean-shutdown flush then wrote it over the correctly restored file: the step
  that exists to protect restored settings was corrupting them instead. Values
  are now quoted and escaped, so commas, quotes, backslashes and parentheses all
  survive exactly as archived, including leading and trailing spaces.
- A restored skin setting whose ID contained a parenthesis or a quote could hang
  a restore. Such an ID collapsed the builtin to its one-argument form, which
  opens an on-screen keyboard and blocks until answered, on a TV with nothing
  expecting input. IDs are now validated and unsafe ones are skipped.
- The hardware-verification gate can no longer be satisfied with stale evidence
  for one device class. Each verified device now records the exact storage-code
  fingerprint it was tested against, instead of the artifact carrying one shared
  value that a later run of a different device would silently re-stamp.

v2026.07.18.0
- Post-restore prompts no longer lose what you typed. A destroyed dialog is
  indistinguishable from a deliberate answer through Kodi's Python API
  (dialog.select returns -1 for both "Back" and "torn down"; the keyboard's
  isConfirmed() is False for both), and the old flow treated a teardown as an
  answer, cleared the once-only marker and moved on. The device-name step now
  re-presents a non-answer instead of advancing, carries half-typed text forward
  into the re-presented keyboard, and only an explicit Keep or a confirmed entry
  ends it. Bounded to five attempts so an unattended box cannot spin.
- Fixed the cause, not just the symptom: the boot service prompted as soon as the
  home window was visible, which is the exact moment skin.estuary7's first-boot
  menu rebuild arms a 15-second timer. That rebuild ends in ReloadSkin(), which
  destroys the whole window stack. Reproduced twice on a local Kodi 21.3 bench at
  15.05s and 15.45s, killing an open prompt 15.27 seconds in. The service now
  waits past that deferred build, and honours skinshortcuts' own in-progress flag,
  before opening anything.
- Version bumped because the storage contract changed. The previous branch state
  carried modified nsud.py/wiz.py while still claiming to be 2026.07.17.7, which
  would have let a device-verification run certify code the box was not actually
  running, since verify_device.py compares version strings.

  residue / dangerous stale keys) and auto-fixes the dangerous class with one silent fresh
  pass. The old mid-flight warning ("could not remove N items; leftovers may shadow...") is

  prompts (this is exactly how a restored Apple TV came back in the stock skin).
