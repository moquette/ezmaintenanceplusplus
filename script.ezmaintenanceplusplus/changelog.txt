2026.04.05.0
 - Updated for Kodi 22

2026.07.15.0
 - First-ever run now offers the same two-step tune-up a restore does: name this
   device, then size the video cache buffer for it. Exactly-once per install;
   boxes that already ran EZ Maintenance++ are not re-prompted by the update.

2026.07.15.1
 - Backup now confirms the final backup name (with its auto timestamp) before it
   starts, matching the restore confirm. Cancelling that prompt writes nothing.
   Applies to local, network, and Dropbox backups.

2026.07.15.2
 - Maintenance: new "Clear All" runs cache, packages, thumbnails, and (if any
   are found) the recently played channels in one action.
 - Maintenance: new "Clear Recently Played Channels" resets the PVR recently-
   played list. The write happens with the PVR manager paused so the running
   client cannot clobber it; it is a no-op with a notice when none are found.

2026.07.15.3
 - Clearing recently played channels now offers a Kodi restart. The home-screen
   widget reads the PVR "last played" list from the running manager's memory, so
   the cleared list only shows after Kodi reloads.

2026.07.15.4
 - Apple TV: the restart prompt now uses the honest "close and reopen Kodi"
   wording on tvOS. It was detected only as non-Android and mislabelled
   "Restart now?", but Kodi cannot self-restart on tvOS (Quit only closes).
   Fixes the wording for the new channel-clear restart offer and for every
   restore / Fresh Start prompt on Apple TV.

2026.07.16.0
 - Full backup now truly means full. The 2026.07.08.5 pvr.iptvsimple exclusion
   is reversed: IPTV settings (instance-settings, settings.xml, custom groups)
   are captured on BOTH Apple TV (from NSUserDefaults) and Fire TV, and a
   restore brings them back exactly as backed up. Stray instance files the
   archive does not carry are swept AFTER the extract (a canceled restore can
   never cost the box the config it already had), with every removal verified.
 - Restores briefly pause the IPTV client for the extract window and always
   re-enable it, so the live client can no longer overwrite the restored IPTV
   settings at shutdown.
 - Every backup embeds backup_manifest.json (what it holds, what it could not
   capture); restore verifies the archive against it. All backup / restore /
   wipe / Fresh Start reporting is now honest: partial results are reported as
   INCOMPLETE with the exact items, never "Complete", and an incomplete backup
   never rotates out the last good one.
 - Wipes on Apple TV clear BOTH storage layers (files and NSUserDefaults keys,
   verified against the store), and a one-time service migration purges stale
   hidden settings copies left by 2026.07.08.x-2026.07.13.x, materializing any
   key-only file to disk first so nothing is ever lost.
 - New Tools menu: "Purge stale tvOS keys" and "Verify backup archive".

2026.07.16.1
 - Verified live on real hardware (Fire TV + Apple TV): full backup captures
   IPTV on both OSes, restore brings it back exactly (strays swept, IPTV client
   paused and resumed), honest reporting confirmed. No functional change from
   2026.07.16.0 - this release records the hardware verification and the move to
   CI-driven publishing.
